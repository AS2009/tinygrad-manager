## Why TinyGrad Manager

Running local AI models typically means juggling multiple CLI tools, shell scripts, and Python environments. TinyGrad Manager wraps everything into a single native Mac app:

- **One-click launch** — no `source venv/bin/activate`, no `uvicorn` commands
- **Menu bar resident** — close the window, the server keeps running
- **OpenAI drop-in** — any app that speaks the OpenAI SDK works out of the box (Cursor, Continue, custom scripts)
- **Full GPU control** — pick which GPU runs your LLM vs. your image model independently

## Features

### Model Management
- Browse and load local model files via GUI file picker
- Supported formats: `.safetensors`, `.pth`/`.pt`, `.gguf`, `.mlx`, `.json`
- Auto-parse weight structure and wire to API converter
- Per-model GPU assignment: LLM and image models each get their own GPU selector

### Text-to-Image
- Stable Diffusion via `diffusers` (HuggingFace)
- Local checkpoint files (`.safetensors`, `.ckpt`, `.pt`, `.bin`) or HuggingFace model IDs
- Customizable prompt, resolution, steps, CFG scale, seed
- Output saved to `~/TinyGradManager/output/`

### GPU Monitoring & Assignment
- Auto-detect system GPU via `system_profiler` + `torch.cuda` + `tinygrad`
- **Metal** (Apple Silicon) / **CUDA** (NVIDIA) support
- eGPU detection via `SafeEjectGPU`
- Independent GPU selectors for LLM and image models

### Background Services
- **GPU Service** — launchctl-managed TinyGrad runtime daemon
- **API Service** — one-click FastAPI + uvicorn, exposing models as HTTP API

### OpenAI-Compatible API

| Endpoint                    | Method | Description                                |
|-----------------------------|--------|--------------------------------------------|
| `/v1/models`                | GET    | List loaded models                         |
| `/v1/chat/completions`      | POST   | Chat completions (supports `stream: true`) |
| `/v1/images/generations`    | POST   | Image generation (DALL·E compatible)       |

Default: `http://localhost:1234`. Drop-in replacement for OpenAI SDK `base_url`.

### Menu Bar
- Close window → hide to menu bar (app stays running, no Dock icon)
- `LSUIElement` keeps the Dock clean
- Menu: Show/Hide window / Quit

### Localization
- English & 中文, auto-detected from system language

### Liquid Glass Design (macOS 26)
- Larger card corner radius (18pt), subtle depth shadows
- Enhanced window translucency with top-down luminance gradient
- Graceful fallback to standard glass on macOS 14–15

## Quick Start

### Download DMG

1. Grab the latest `.dmg` from [Releases](https://github.com/AS2009/tinygrad-manager/releases)
2. Mount and drag `TinyGradManager.app` to `/Applications`
3. Launch — the menu bar icon appears, backend starts automatically
4. Send a test request:

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:1234/v1",
    api_key="local"  # any value works — no auth required
)

print(client.models.list())
```

### Run from source

```bash
# 1. Install Python dependencies
pip install -r requirements.txt

# 2. Start the Python backend
python TinyGradManager/backend_main.py --port 1234 &

# 3. Build and run the Swift GUI
cd SwiftGUI && swift run
```

### Build .app bundle locally

```bash
cd SwiftGUI
swift build --configuration release --arch arm64
make app
# → TinyGradManager.app
```

The CI workflow (`.github/workflows/main.yml`) handles full `.app` → `.dmg` assembly automatically.

### API Usage

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:1234/v1",
    api_key="local"
)

# Chat completion (streaming)
response = client.chat.completions.create(
    model="your-model-name",
    messages=[{"role": "user", "content": "Hello!"}],
    stream=True
)
for chunk in response:
    print(chunk.choices[0].delta.content, end="")

# Image generation (DALL·E compatible)
image = client.images.generate(
    model="stable-diffusion-v1-5",
    prompt="a cat sitting on a stack of GPUs",
    size="512x512",
    n=1
)
```

### Swift source (`SwiftGUI/Sources/`)

| File                     | Purpose                                                |
|--------------------------|--------------------------------------------------------|
| `TinyGradManagerApp.swift` | @main entry, NSApplicationDelegate, menu bar, window |
| `ContentView.swift`      | Root layout, header with connection status indicator   |
| `BackendClient.swift`    | HTTP client, @Observable state, polling, error tracking|
| `BackendManager.swift`   | Python subprocess lifecycle, crash recovery, stdout capture |
| `Models.swift`           | Codable types, ConnectionState / ProcessState enums    |
| `L10n.swift`             | English / Chinese localization                         |

| `ModelCard.swift`        | Card 1 — model file browser + GPU picker + load        |

| `SystemStatusCard.swift` | Card 3 — GPU info, GPU Service toggle, env check       |
| `ConsoleCard.swift`      | Card 4 — scrollable log output with auto-scroll        |

### Runtime flow

1. Swift binary launches → creates `BackendManager`
2. `BackendManager` starts `python3 backend_main.py --port 1234` as child process
3. `BackendClient` polls `/api/status` every 2s until backend is ready
4. All user actions go through HTTP to the Python backend
5. On quit: SIGTERM → wait 3s → SIGKILL for clean shutdown
6. On crash: up to 3 automatic restart attempts

## Troubleshooting

### GPU not detected
```bash
# Check Metal support
system_profiler SPDisplaysDataType | grep -E "Chipset Model|Metal"

# Check tinygrad
python -c "from tinygrad import Device; print(list(Device.get_available_devices()))"

# Check torch MPS
python -c "import torch; print('MPS available:', torch.backends.mps.is_available())"
```

### Image generation is slow on first run
- First run downloads ~5 GB from HuggingFace — this is normal, one-time only
- If MPS runs out of memory, try a smaller resolution (e.g. 384×384)

### Backend won't start
- Ensure port 1234 is not already in use: `lsof -i :1234`
- Check the Console card in the app for error logs
- The backend auto-restarts up to 3 times on crash

## 为什么选择 TinyGrad 管理器

在本地运行 AI 模型通常意味着需要同时操作多个命令行工具、Shell 脚本和 Python 环境。TinyGrad 管理器将一切整合到单个原生 Mac 应用中：

- **一键启动** — 无需 `source venv/bin/activate`，无需手动执行 `uvicorn`
- **菜单栏驻留** — 关闭窗口后，后台服务继续运行
- **OpenAI 即插即用** — 任何使用 OpenAI SDK 的应用都能开箱即用（Cursor、Continue、自定义脚本等）
- **完整 GPU 控制** — 为 LLM 和图像模型各自独立分配 GPU

## 功能特性

### 模型管理
- 通过 GUI 文件选择器浏览和加载本地模型文件
- 支持格式：`.safetensors`、`.pth`/`.pt`、`.gguf`、`.mlx`、`.json`
- 自动解析权重结构并接入 API 转换器
- 独立 GPU 分配：LLM 和图像模型各自拥有独立的 GPU 选择器

### 文生图
- 基于 `diffusers` 的 Stable Diffusion（HuggingFace）
- 支持本地模型文件（`.safetensors`、`.ckpt`、`.pt`、`.bin`）或 HuggingFace 模型 ID
- 可自定义提示词、分辨率、步数、CFG scale、种子
- 输出保存至 `~/TinyGradManager/output/`

### GPU 监控与分配
- 启动时自动检测系统 GPU（`system_profiler` + `torch.cuda` + `tinygrad`）
- **Metal**（Apple Silicon 内置 GPU）/ **CUDA**（NVIDIA）支持
- 通过 `SafeEjectGPU` 检测 eGPU
- LLM 和图像模型各自独立的 GPU 选择器

### 后台服务
- **GPU 服务** — launchctl 管理的 TinyGrad 运行时守护进程
- **API 服务** — 一键启动 FastAPI + uvicorn，将模型暴露为 HTTP API

### OpenAI 兼容 API

| 端点                       | 方法   | 描述                               |
|----------------------------|--------|------------------------------------|
| `/v1/models`               | GET    | 列出已加载的模型                    |
| `/v1/chat/completions`     | POST   | 聊天补全（支持 `stream: true`）      |
| `/v1/images/generations`   | POST   | 图像生成（兼容 DALL·E API）         |

默认地址：`http://localhost:1234`。可直接替换 OpenAI SDK 的 `base_url`。

### 菜单栏驻留
- 关闭窗口 → 隐藏至菜单栏（应用保持运行，无 Dock 图标）
- `LSUIElement` 保持 Dock 栏整洁
- 菜单：显示/隐藏窗口 / 退出

### 语言支持
- English & 中文，根据系统语言自动切换

### Liquid Glass 设计（macOS 26）
- 更大的卡片圆角（18pt），微妙的深度阴影
- 增强的窗口半透明效果，顶部渐变光效
- macOS 14–15 优雅降级至标准玻璃效果

## 常见问题

### 内置 GPU 未显示
```bash
# 检查 Metal 支持
system_profiler SPDisplaysDataType | grep -E "Chipset Model|Metal"

# 检查 tinygrad
python -c "from tinygrad import Device; print(list(Device.get_available_devices()))"

# 检查 torch MPS
python -c "import torch; print('MPS 可用:', torch.backends.mps.is_available())"
```

### 文生图首次运行速度慢
- 首次运行需从 HuggingFace 下载约 5 GB 模型文件，仅此一次
- 如果 MPS 内存不足，可尝试使用更低的分辨率（如 384×384）

### 后端无法启动
- 确保端口 1234 未被占用：`lsof -i :1234`
- 在应用的 Console 卡片中查看错误日志
- 后端崩溃后会自动重启，最多重试 3 次
