<h2 id="english">TinyGrad Manager</h2>

> Native macOS menu bar app with **Apple Liquid Glass** design. Manage [TinyGrad](https://github.com/tinygrad/tinygrad) models, text-to-image generation, GPU services, and expose an OpenAI-compatible HTTP API.

Built with **SwiftUI** + embedded Python backend. Fully self-contained — no manual setup required.

---

## Features

### Model Management
- Browse and load local model files via GUI
- Supported formats: `.safetensors`, `.pth`/`.pt`, `.gguf`, `.mlx`, `.json`
- Auto-parse weight structure and wire to API converter
- **Per-model GPU assignment**: LLM and image models each get their own GPU selector

### Text-to-Image
- Stable Diffusion via `diffusers` (HuggingFace)
- Local checkpoint files (`.safetensors`, `.ckpt`, `.pt`, `.bin`) or HuggingFace model IDs
- Customizable prompt, resolution, steps, CFG scale, seed
- Output saved to `~/TinyGradManager/output/`

### GPU Monitoring
- Auto-detect system GPU via `system_profiler` + `torch.cuda` + `tinygrad`
- **Metal** (Apple Silicon) / **CUDA** (NVIDIA) support
- eGPU detection via `SafeEjectGPU`
- Independent GPU selectors for LLM and image models

### Services
- **GPU Service**: launchctl-managed TinyGrad runtime daemon
- **API Service**: One-click FastAPI + uvicorn, exposing models as HTTP API

### OpenAI-Compatible API

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/v1/models` | GET | List loaded models |
| `/v1/chat/completions` | POST | Chat completions (supports `stream: true`) |
| `/v1/images/generations` | POST | Image generation (DALL·E compatible) |

Default: `http://localhost:1234`. Drop-in replacement for OpenAI SDK `base_url`.

### Menu Bar
- Close window → hide to menu bar (app stays running)
- No Dock icon (`LSUIElement`)
- Menu: Show/Hide window / Quit

### Localization
- English & 中文 (auto-detected from system language)

### Liquid Glass Design (macOS 26)
- Larger card corner radius (18pt)
- Subtle depth shadows on cards
- Enhanced window translucency
- Top-down luminance gradient in header
- Graceful fallback to standard glass on macOS 14–15

**Runtime flow:**
1. Swift binary launches → creates `BackendManager`
2. `BackendManager` starts `python3 backend_main.py --port 1234` as child process
3. `BackendClient` polls `/api/status` every 2s until backend is ready
4. All user actions go through HTTP to the Python backend
5. On quit: SIGTERM → wait 3s → SIGKILL for clean shutdown
6. On crash: up to 3 automatic restart attempts

## Quick Start

### Run from source
```bash
# Install Python dependencies
pip install -r requirements.txt

# Start the Python backend
python TinyGradManager/backend_main.py --port 1234 &

# Build and run the Swift GUI
cd SwiftGUI && swift run
```

### Build .app bundle (local)
```bash
# Build Swift binary
cd SwiftGUI && swift build --configuration release --arch arm64

# The CI workflow (main.yml) handles full .app assembly:
#   Swift binary + Python backend + site-packages → .app → DMG
```

### API Usage
```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:1234/v1",
    api_key="not-needed"
)

response = client.chat.completions.create(
    model="your-model-name",
    messages=[{"role": "user", "content": "Hello!"}],
    stream=True
)
for chunk in response:
    print(chunk.choices[0].delta.content, end="")
```

## Troubleshooting

### GPU not showing up
```bash
# Check Metal support
system_profiler SPDisplaysDataType | grep -E "Chipset Model|Metal"

# Check tinygrad
python -c "from tinygrad import Device; print(list(Device.get_available_devices()))"

# Check torch MPS
python -c "import torch; print('MPS available:', torch.backends.mps.is_available())"
```

### Image generation slow
- First run downloads ~5 GB from HuggingFace — this is normal
- If MPS runs out of memory, try a smaller resolution (384×384)
